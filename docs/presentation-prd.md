## 1. 제품 개요

### 한 줄 소개
**스마트폰 영상 한 편으로 14가지 보행 지표를 자동 분석하는 AI 보행분석 시스템**

### 제품명
AI Gait Analysis System (AI 보행분석 시스템)

### 기반 검사
**10-Meter Walk Test (10MWT)** — 전 세계적으로 가장 널리 사용되는 표준 보행 평가 도구

### 핵심 가치
| 기존 방식 | AI 보행분석 시스템 |
|-----------|-------------------|
| 스톱워치로 시간만 측정 | 14가지 정밀 보행 지표 자동 측정 |
| 검사자 숙련도에 따라 결과 편차 | AI가 일관된 기준으로 분석 |
| 보행속도 1가지만 기록 | 속도, 보폭, 활보장, 좌우 대칭성, 스윙/스탠스 비율 등 |
| 고가 장비(압력판, 센서) 필요 | 스마트폰 카메라 1대면 충분 |
| 결과 해석에 전문 지식 필요 | 자동 판정 + 임상 코멘트 제공 |
| 기록 보관 어려움 | 환자별 이력 관리 + 추이 그래프 |

- 정기적 보행 능력 모니터링

## 3. 핵심 기능 상세

### 3-1. AI 영상 분석 파이프라인

```
[영상 업로드] → [2-Pass 분석] → [자동 판정] → [리포트 생성]
```

#### Pass 1: 캘리브레이션 (거리 보정)
- **입력**: 환자 키(cm)
- **방법**: AI가 영상 속 사람의 키 픽셀을 자동 측정
- **결과**: PPM(Pixels Per Meter) 산출 → 모든 거리 측정의 기준
- **원근 보정**: 카메라와의 거리에 따라 프레임마다 동적으로 PPM 조정
- **의미**: 고가의 거리 측정 장비 없이도 cm 단위 정밀 거리 측정 가능

#### Pass 2: 보행 분석
- **자세 추정**: YOLOv8-Pose + MediaPipe 듀얼 AI 모델
  - YOLOv8: 실시간 사람 감지 및 17개 관절 추정 (NVIDIA GPU 가속)
  - MediaPipe Pose Heavy: 33개 관절 정밀 추정 (발꿈치/발끝 포함)
- **보행 구간 자동 감지**: 전신이 화면에 보이는 순간 자동 시작/종료
- **Heel Strike 감지**: 3가지 독립 알고리즘 → 품질 기반 자동 선택
  - Y-peak (발목 수직 위치 피크)
  - Vx-crossing (발목 속도 전환점)
  - Foot Separation (양발 간격 변화)
- **처리 성능**: GPU 가속 시 3~5 fps 실시간 분석

### 3-2. 14가지 정밀 보행 지표

#### 보행 지표 (2개)
| 지표 | 설명 | 정상 범위 | 임상 의미 |
|------|------|-----------|-----------|
| **보행속도** | 단위 시간당 이동 거리 | 1.0~1.4 m/s | 전반적 보행 능력의 핵심 지표 ("제6의 활력징후") |
| **분속수 (Cadence)** | 1분당 걸음 수 | 100~120 steps/min | 보행 리듬 및 자동화 수준 |

#### 거리 변수 (4개) — 좌/우 개별 측정
| 지표 | 설명 | 정상 범위 | 임상 의미 |
|------|------|-----------|-----------|
| **좌측 보폭 (L Step)** | 오른발→왼발 착지 거리 | 55~70 cm | 좌측 하지 추진력 |
| **우측 보폭 (R Step)** | 왼발→오른발 착지 거리 | 55~70 cm | 우측 하지 추진력 |
| **좌측 활보장 (L Stride)** | 왼발→왼발 1보행주기 거리 | 110~140 cm | 좌측 보행주기 완성도 |
| **우측 활보장 (R Stride)** | 오른발→오른발 1보행주기 거리 | 110~140 cm | 우측 보행주기 완성도 |

#### 시간 변수 (4개)
| 지표 | 설명 | 정상 범위 | 임상 의미 |
|------|------|-----------|-----------|
| **좌측 스텝 시간** | 반대발→좌발 착지 소요 시간 | 0.50~0.60 s | 좌측 타이밍 조절 |
| **우측 스텝 시간** | 반대발→우발 착지 소요 시간 | 0.50~0.60 s | 우측 타이밍 조절 |
| **좌측 활보 시간** | 좌발 1보행주기 소요 시간 | 1.00~1.20 s | 좌측 보행 주기 |
| **우측 활보 시간** | 우발 1보행주기 소요 시간 | 1.00~1.20 s | 우측 보행 주기 |

#### 비율 변수 (4개)
| 지표 | 설명 | 정상 범위 | 임상 의미 |
|------|------|-----------|-----------|
| **입각기 비율 (Stance)** | 발이 바닥에 접촉한 비율 | 58~62% | 안정성 지표 |
| **유각기 비율 (Swing)** | 발이 공중에 있는 비율 | 38~42% | 추진력 지표 |
| **스윙/스탠스 비율** | Swing ÷ Stance | ~0.67 | 보행 효율성 |
| **좌우 대칭성 (SI)** | 좌우 차이 비율 | < 10% | 비대칭 = 병리적 보행 |

### 3-3. 자동 판정 시스템

#### Perry 보행 속도 분류 (Perry et al., 1995)
전 세계적으로 가장 널리 인용되는 보행 속도 분류 체계:

| 분류 | 속도 기준 | 임상 의미 |
|------|-----------|-----------|
| Household Ambulator | < 0.4 m/s | 실내 보행만 가능, 높은 의존도 |
| Limited Community | 0.4~0.8 m/s | 제한적 외출, 보조 필요 |
| Full Community | 0.8~1.0 m/s | 독립적 외출 가능 |
| Normal Speed | 1.0~1.4 m/s | 정상 보행 능력 |

#### 변수별 정상범위 판정
- 각 변수를 한국 성인 기준 정상범위와 비교
- **정상** / **상한 초과(↑)** / **하한 미달(↓)** 자동 분류
- 편차율(%) 자동 계산 → 얼마나 벗어났는지 정량화

#### 보행 패턴 자동 감지 (3가지)
| 패턴 | 감지 조건 | 임상 의미 |
|------|-----------|-----------|
| **안정성 보상 패턴** | ↑입각기 + ↓유각기 + ↓속도 | 균형 불안 시 보이는 보상 전략 |
| **단보 패턴** | ↓보폭 + ↓활보장 + ↑분속수 | 한국 고령자에게 흔한 패턴 (Kim et al., 2024) |
| **높은 변동성** | SI ≥ 10% | 좌우 비대칭 → 편측 약화 의심 |

#### 대칭성 지수 (Symmetry Index)
- 공식: `SI = |좌-우| / (0.5 × (좌+우)) × 100%`
- 기준: **< 10% = 정상** (Patterson et al., 2010; Herzog et al., 1989)
- 보폭, 활보장, 스텝시간, 활보시간, 스윙/스탠스 6가지 항목 개별 SI 산출
- 종합 대칭성 지수(Overall SI)도 자동 계산

### 3-4. 근거 영상 클립 (Evidence Clips)
- 각 측정 변수마다 **해당 수치가 측정된 영상 구간**을 자동 추출
- 평균값에 가장 가까운 대표 스텝을 자동 선택하여 재생
- 영상 위에 실시간 오버레이:
  - 발목 위치 추적 (좌: 파란색, 우: 빨간색)
  - Heel Strike 마커 + 측정선
  - 실시간 거리/시간 카운터
  - 판정 게이지 바
- **의미**: "이 수치가 어떻게 나왔는지" 영상으로 직접 확인 → **기록의 신뢰도 극대화**

### 3-5. 분석 영상 플레이어
- 전체 보행 구간 영상 재생 + 실시간 오버레이
- HUD 표시: 속도, 케이던스, 누적 스텝 수, 경과 시간
- 프레임 단위 앞뒤 이동 가능
- 재생 속도 조절 (0.25x ~ 2x)
- 타임라인 스크러버로 원하는 시점 즉시 이동

### 3-6. 환자 관리 및 이력 추적
- 환자 프로필 등록 (이름, 생년월일, 진단명, 키)
- 검사 이력 자동 저장 및 조회
- **추이 그래프**: 시간에 따른 보행속도 변화 시각화
- **검사 비교**: 2개 검사 결과를 나란히 비교 → 치료 효과 객관적 확인
- **대시보드**: 총 환자 수, 오늘 검사 수, 주간 통계, 진단별 분포

## 4. 기존 방식과의 비교

### 4-1. vs 스톱워치 10MWT

| 항목 | 스톱워치 | AI 보행분석 |
|------|----------|-------------|
| 측정 지표 | 보행속도 1가지 | 14가지 지표 + 대칭성 + 패턴 |
| 측정 정밀도 | 검사자 반응시간 오차 (±0.2~0.5s) | 프레임 단위 정밀도 (±0.017s @60fps) |
| 좌우 분리 | 불가 | 좌/우 개별 측정 + 대칭성 지수 |
| 재현성 | 검사자 간 편차 존재 | 동일 영상 → 동일 결과 |
| 기록 보존 | 종이 기록, 분실 위험 | 디지털 이력 관리 + 영상 보존 |
| 근거 확인 | 불가 (기억에 의존) | 영상 클립으로 즉시 확인 |
| 소요 시간 | ~5분 (세팅 + 3회 시행) | ~2분 (영상 촬영 1회 + 자동 분석) |

### 4-2. vs 고가 보행분석 장비

| 항목 | GAITRite / Vicon 등 | AI 보행분석 |
|------|---------------------|-------------|
| 장비 비용 | 3,000만원~1억원+ | 스마트폰 + PC (기존 장비 활용) |
| 설치 공간 | 전용 보행로 필요 (6~10m) | 일반 복도에서 촬영 가능 |
| 측정 지표 | 30+ 정밀 지표 | 14가지 핵심 임상 지표 |
| 측정 정밀도 | 매우 높음 (mm 단위) | 높음 (cm 단위, 키 기반 보정) |
| 임상 활용도 | 연구/대형병원 위주 | 1차 의료기관/요양시설까지 확대 가능 |
| 접근성 | 장비 보유 기관만 | 인터넷 연결된 곳이면 어디서나 |

### 4-3. 핵심 차별점 요약

1. **진입 장벽 제로**: 스마트폰 카메라 1대 + 환자 키 정보만 있으면 시작
2. **임상 수준의 정밀도**: 듀얼 AI 모델 + 동적 원근 보정 + 다중 HS 감지 알고리즘
3. **투명한 근거**: 모든 수치에 대한 영상 증거 클립 자동 생성
4. **즉각적 임상 해석**: Perry 분류 + 정상범위 판정 + 패턴 감지 + 임상 코멘트
5. **이력 추적**: 환자별 시계열 데이터 → 치료 효과의 객관적 입증

## 5. 기술 신뢰성

### 5-1. 사용된 AI 모델
| 모델 | 용도 | 특징 |
|------|------|------|
| **YOLOv8n-Pose** | 실시간 사람 감지 + 17개 관절 추정 | Ultralytics, COCO 학습, GPU 가속 |
| **MediaPipe Pose Heavy** | 33개 관절 정밀 추정 | Google, 발꿈치/발끝 감지 포함 |

### 5-2. 거리 보정 방법
- **키 기반 캘리브레이션**: 환자의 실제 키 → 영상 속 키 픽셀 비교 → PPM 산출
- **동적 원근 보정**: 카메라에서 멀어질수록 사람이 작아지는 효과를 프레임마다 보정
- 별도 마커나 장비 불필요

### 5-3. Heel Strike 감지 정확도
- **3중 독립 알고리즘** → 품질 점수 기반 자동 선택
- 감도 3단계 적응형 조절 (prominence: 0.06 → 0.03 → 0.015)
- Peak deceleration 모드: 최대 감속점 기반 → 실제 Initial Contact에 가장 근접
- **검증 결과**: 7개 테스트 영상에서 TimeSI 평균 7% (수용 가능 수준)

### 5-4. 판정 기준 학술 근거
| 기준 | 출처 |
|------|------|
| 보행속도 분류 | Perry J et al. (1995) Classification of Walking Handicap |
| 대칭성 지수 10% 기준 | Patterson KK et al. (2010); Herzog W et al. (1989) |
| 정상 보행 범위 | AAFP (2010) Gait Disorders in Older Adults |
| 낙상 위험 속도 기준 | Verghese J et al. (2009) Quantitative gait dysfunction |
| 단보 패턴 한국 고령자 | Kim YH et al. (2024) Gait characteristics in Korean elderly |
| 장애 위험 속도 기준 | Cesari M et al. (2005) Prognostic value of usual gait speed |

## 6. 사용 워크플로우

### 촬영부터 결과까지 4단계

```
[1단계] 검사 설정
  └─ 환자 선택 + 키 입력 + 거리 설정 (기본 10m)

[2단계] 영상 촬영 & 업로드
  └─ 스마트폰으로 측면에서 보행 영상 촬영 → 업로드
  └─ 촬영 가이드: 측면 90°, 전신 포함, 10m 직선 보행

[3단계] AI 자동 분석 (약 1~2분)
  └─ Pass 1: 키 캘리브레이션 (거리 보정)
  └─ Pass 2: 보행 분석 (Heel Strike 감지, 거리/시간 측정)
  └─ 실시간 진행률 표시 + 라이브 프레임 미리보기

[4단계] 결과 확인
  └─ 보행속도 Perry 분류
  └─ 14가지 변수 판정표 (정상/초과/미달 + 편차%)
  └─ 좌우 대칭성 분석 (6가지 SI)
  └─ 보행 패턴 감지
  └─ 영상 클립 근거 확인
  └─ 임상 코멘트 + 권고사항
```

## 7. 제품 장단점 (투명한 공개)

### 장점
1. **경제성**: 스마트폰 + PC만으로 임상급 보행분석 가능 (고가 장비 불필요)
2. **객관성**: AI 기반 일관된 분석 → 검사자 간 변동성 제거
3. **포괄성**: 1회 촬영으로 14가지 지표 동시 측정
4. **투명성**: 영상 근거 클립으로 모든 수치의 출처 확인 가능
5. **편의성**: 촬영 → 업로드 → 2분 내 결과 완성
6. **추적성**: 환자별 이력 관리 + 추이 그래프로 치료 효과 시각화
7. **접근성**: 대형 병원뿐 아니라 1차 의료기관, 요양시설에서도 활용 가능
8. **학술 근거**: 모든 판정 기준에 국제 학술지 참고문헌 명시

### 한계 및 보완 계획
1. **측면 촬영 전용**: 현재 측면 카메라 위치에서만 정확한 측정 (전면/후면 미지원)
   - → 향후 다중 카메라 앵글 지원 예정
2. **거리 정밀도**: 키 기반 보정은 mm 단위 장비 대비 ~5-10% 오차 가능
   - → 연구용이 아닌 임상 스크리닝 수준에서 충분한 정밀도
3. **연령/속도별 정상범위**: 현재 성인 일반 기준 적용 중
   - → 연령대/성별/진단별 세분화된 정상범위 DB 구축 예정
4. **인터넷 필요**: 현재 로컬 서버 기반 (오프라인 모드 개발 중)
5. **30fps 이상 권장**: 낮은 프레임률 영상에서는 Heel Strike 감지 정밀도 저하
   - → 대부분 스마트폰이 30fps 이상 지원하므로 실사용에는 문제 없음

> 80세 여성, 최근 보행 불안정 호소.

> - 자동 알림: "낙상 위험 1.5배 증가 (Verghese 2009)"

## 11. 핵심 메시지 (PPT 슬라이드용)

### 슬라이드 1: Hook
> "스톱워치 하나로 보행을 평가하던 시대는 끝났습니다."

### 슬라이드 2: Problem
> 기존 10MWT는 **보행속도 1가지만** 측정합니다.
> 좌우 차이, 보폭 변화, 보행 패턴은 보이지 않습니다.
> 고가 장비는 대형 병원에만 있고, 정작 필요한 1차 의료기관에는 없습니다.

### 슬라이드 3: Solution
> 스마트폰 영상 1편 → AI가 **14가지 보행 지표** 자동 분석
> 모든 수치에 **영상 근거** 첨부 → 기록의 신뢰도 보장

### 슬라이드 4: How It Works
> 4단계: 설정 → 촬영 → AI 분석 (2분) → 결과 확인
> Perry 분류 + 정상범위 판정 + 패턴 감지 + 임상 코멘트

### 슬라이드 5: Evidence & Trust
> - 국제 학술지 기반 판정 기준 (Perry 1995, Patterson 2010, Verghese 2009)
> - 모든 측정값에 영상 클립 근거 자동 첨부
> - 좌우 개별 측정 + 대칭성 지수 → 편측 약화 조기 발견

### 슬라이드 6: Comparison
> | | 스톱워치 | 고가 장비 | AI 보행분석 |
> |---|---------|----------|------------|
> | 비용 | 0원 | 3,000만원+ | 기존 장비 활용 |
> | 지표 | 1개 | 30+개 | 14개 (임상 핵심) |
> | 접근성 | 높음 | 매우 낮음 | 높음 |
> | 근거 | 없음 | 있음 | 영상 클립 |

### 슬라이드 7: Use Cases
> 뇌졸중 재활 추적 / 낙상 위험 스크리닝 / 수술 후 복귀 평가

### 슬라이드 8: Call to Action
